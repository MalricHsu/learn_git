# Git Daily 中文化、重玩與練習題庫設計

## 目標

降低中英文並列造成的閱讀負擔、讓已完成關卡可真正重玩、修正清除瀏覽器資料後仍帶有示範進度的問題，並把兩種練習模式各擴充到 60 題且提供不洩漏答案的提示與答錯解釋。

## 手冊中文化

- 手冊文章的說明型標題統一只顯示中文，例如「什麼是這個指令」、「為什麼需要它」、「工作流程」、「語法與範例」、「常見錯誤」、「相關指令」及「Git 專有名詞」。
- 說明段落只保留繁體中文，不再於下一行重複英文翻譯。
- 文章名稱、Git 指令、參數、終端輸出及無法合理翻譯的 Git 術語保留英文。
- 側欄分類與必要的操作標籤可保留簡短英文，但不得再次呈現一整段相同意思的雙語內容。
- 全面檢查目前 Claude Code 整理到一半的 `Reference.vue`，避免只修正畫面截圖中的兩個區塊。

## 闖關重玩與進度

- 永久完成紀錄只負責顯示勾選、解鎖後續關卡及避免重複取得 XP。
- 每次進入任何已完成關卡時，都建立新的本次作答狀態：清除選項、答案、完成卡、終端紀錄與工作流程勾選。
- 已完成關卡仍可正常選擇答案或輸入指令；只有本次再次答對後才顯示完成回饋。
- 重玩答對不重複增加 XP，也不重複寫入完成紀錄。
- 新使用者或清空 localStorage 後的預設狀態為 0 XP、0 個完成關卡、空白收藏與空白最近閱讀紀錄。主題設定獨立保留既有行為。

## 練習題庫與提示

- 選擇題題庫由 35 題擴充為剛好 60 題。
- 手寫題題庫由 40 題擴充為剛好 60 題。
- 題目可同時支援兩種模式；最終以 `poolFor("choice")` 與 `poolFor("input")` 的實際數量為準。
- 新題涵蓋初始化、暫存、提交、歷史、分支、遠端、復原與進階操作，難度維持現有 L1 至 L3 分布。
- 每題具有繁體中文 `hint`。作答前可展開或收起；提示只提供判斷方向、相關概念或指令結構，不顯示完整正解。
- 答錯後立即顯示使用者答案、正確答案及該題的繁體中文解釋；既有 `note.zh` 可作為解釋，缺乏針對性的題目需補強。
- 答對後仍可顯示精簡中文補充，但不重複英文解說。

## 資料與元件邊界

- `src/data/drills.js`：題目、答案規則、提示與答後解釋。
- `src/views/Practice.vue`：提示的開關狀態、答錯回饋與題目呈現。
- `src/views/Game.vue`：區分永久完成與本次完成，控制重玩狀態。
- `src/composables/useStore.js`：乾淨的新使用者預設值與持久化。
- `src/views/Reference.vue`：手冊說明型內容中文化。

## 驗證

- 自動測試確認兩個題庫數量皆為 60、每題 ID 唯一、每題皆有非空中文提示、手寫題皆有可判定答案的規則。
- 元件或純邏輯測試確認已完成關卡仍可作答、重玩不增加 XP，以及空儲存狀態從零開始。
- 建置測試確認 Vue 模板與新增資料可正常編譯。
- 瀏覽器檢查手冊、闖關及兩種練習流程，包含提示展開、答錯解釋與重新整理後的持久化。

## 不在本次範圍

- 不翻譯 Git 指令本身、終端輸出或程式碼。
- 不改變每回合隨機抽 10 題的規則。
- 不讓重玩成為重複刷取 XP 的方式。
