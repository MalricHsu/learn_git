# Git Daily Interactive Front Page Design

## Goal

首頁改為一張可互動的 Git 報紙頭版，讓訪客先理解品牌，再以單一明確入口繼續闖關。

## Structure

1. Masthead：期號、日期、Git Daily 品牌。
2. Front Page：左側主故事與開始學習，右側為無卡片化的 Continue Reading／繼續闖關。
3. Today's Edition：今日焦點指令、編輯精選、今日指令，使用報紙欄線而非 Dashboard cards。
4. Chapter Editions：只顯示六個章節，不顯示 32 張關卡卡片。
5. Headlines：三篇延伸閱讀與完整手冊入口。

## Visual Rules

- 使用粗細橫線、直欄線、大型 serif 標題與大量留白。
- 不使用 XP 圓環、圓角資訊卡或 Dashboard 統計模組。
- 進度只用一條水平 progress 和文字呈現。
- CTA 以「開始／繼續闖關」為第一優先，「翻閱手冊」為第二優先。
- 900px 以下改為單欄，保留報紙閱讀順序。

## Data

- 沿用 store、challenges、chapters、commands。
- 不改進度、XP 或路由邏輯。
